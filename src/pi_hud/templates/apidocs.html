Field reference

FieldRequiredWhat it does / where it shows up
sourceyesApp name (max 64). Message list, detail page, and the alert footer when there is no DNS metadata.
typeyesOne of success info note warning caution error critical. Colors the rail, type label, and severity chip. Groups the pending-queue screen (error/critical → Error, warning/caution → Power, rest → Info).
titleyesHeadline on the panel (max 48, clamped with …) and in all lists.
messagenoBody (max 500). Up to two short lines on the panel; full text on the detail page.
pinnednoDefault false. Active API messages are shown either way; true adds the PIN chip and protects the alert from automatic retention cleanup until it is cleared. Power alerts use the separate Power dip screen setting.
priorityno1–10 (default 5). Highest-priority active message is shown; ties go to oldest. Multiple actives → grouped pending screen.
categorynoFree grouping tag (max 32) — e.g. dns, backup. power is used internally for power alerts.
metadatanoAny JSON object, ≤4096 chars serialized. Shown raw on the detail page. The keys below are optional and get special rendering.
metadata.hostnoFooter label on the alert screen (hostname, device, share name…) + detail field.
metadata.previous_value / updated_valuenoWhen both are set, the panel body becomes a change line old → new, and the dashboard shows Previous/Updated stats. Works for DNS records, IPs, versions, config values…
metadata.record_typenoOptional label prefixed to the change line, e.g. A old → new.
metadata.timelinenoList of {"time","label","detail"} — rendered as the timeline on the detail page.
