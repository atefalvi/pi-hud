Integration

API

Apps authenticate with a bearer token and POST messages. Create a token on the Tokens page.

Same-Pi request (pi-dns-sync)

curl -X POST http://127.0.0.1:{{ port }}/api/v1/messages \ -H "Authorization: Bearer ph_xxx" \ -H "Content-Type: application/json" \ -d '{ "source": "pi-dns-sync", "type": "success", "title": "DNS Updated", "message": "Cloudflare A record updated.", "pinned": true, "priority": 6, "category": "dns", "metadata": { "host": "home.example.com", "record_type": "A", "previous_value": "203.0.113.42", "updated_value": "198.51.100.27" } }'

LAN device request

curl -X POST http://<PI_IP>:{{ port }}/api/v1/messages \ -H "Authorization: Bearer ph_xxx" \ -H "Content-Type: application/json" \ -d '{"source":"nas-backup","type":"success","title":"Backup Done","message":"TrueNAS sync finished","priority":3}'

LAN requests require api_host = 0.0.0.0 in the config and a service restart.

Warning message

curl -X POST http://127.0.0.1:{{ port }}/api/v1/messages \ -H "Authorization: Bearer ph_xxx" -H "Content-Type: application/json" \ -d '{"source":"pi-dns-sync","type":"error","title":"DNS Failed","message":"Cloudflare API rejected update request.","pinned":true,"priority":9}'

Clear the current alert

curl -X POST http://127.0.0.1:{{ port }}/api/v1/messages/current/clear

Health check: GET /health · List: GET /api/v1/messages?status=active
